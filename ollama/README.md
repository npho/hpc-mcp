# Ollama on Apptainer with Open WebUI

This folder provides an Apptainer definition and Open OnDemand app scaffold for an Ollama-backed chat experience with Open WebUI as the front end.

## Files

- ollama.def: Apptainer definition (Rocky Linux minimal + Ollama + uv-based Open WebUI install)
- Makefile: helper build targets
- ood/: Open OnDemand app files

## Prerequisites

- Apptainer installed on the build/run host
- fakeroot configured if building unprivileged

Notes:

- uv is installed inside the container at build time for Open WebUI.
- Ollama is installed inside the same container and serves models locally on port 11434 for the bundled Open WebUI instance.

## Build image

From this folder:

### Unprivileged (fakeroot)

```bash
apptainer build --fakeroot ollama.sif ollama.def
```

### Privileged (sudo/root)

```bash
sudo apptainer build ollama.sif ollama.def
```

### Make targets

```bash
make build
make build DEF_FILE=ollama.def OUT_FILE=ollama.sif

# Optional explicit variants
make build-fakeroot
make build-root
```

## Local run example (with persistent model and UI data)

```bash
mkdir -p "${HOME}/.local/share/ollama/models"
mkdir -p "${HOME}/.local/share/ollama/open-webui"

apptainer run \
  --cleanenv \
  --bind "${HOME}/.local/share/ollama/models:/var/lib/ollama" \
  --bind "${HOME}/.local/share/ollama/open-webui:/var/lib/open-webui" \
  --env OPEN_WEBUI_HOST=0.0.0.0 \
  --env OPEN_WEBUI_PORT=8080 \
  --env OLLAMA_HOST=127.0.0.1:11434 \
  --env OLLAMA_BASE_URL=http://127.0.0.1:11434 \
  ./ollama.sif
```

Then open http://localhost:8080 in your browser.

To preload a model after the container is built, run a one-off command such as:

```bash
apptainer exec \
  --cleanenv \
  --bind "${HOME}/.local/share/ollama/models:/var/lib/ollama" \
  ./ollama.sif \
   bash -lc 'ollama serve >/tmp/ollama.log 2>&1 & sleep 3 && ollama pull llama3.2'
```

## Open OnDemand deployment

1. Copy the ood app folder into a system or dev app location:
   - System-wide: /var/www/ood/apps/sys/ollama
   - User dev app: ~/ondemand/dev/ollama
2. Ensure the app structure is preserved:
   - manifest.yml
   - form.yml
   - submit.yml.erb
   - view.html.erb
   - template/before.sh.erb
   - template/script.sh.erb
3. Customize form defaults in form.yml:
   - cluster
   - image_path (absolute path to ollama.sif)
   - ollama_data_dir (persistent model directory)
   - webui_data_dir (persistent Open WebUI data directory)
   - resource defaults (hours/cores/memory)
4. Update scheduler directives in submit.yml.erb to match your site policy.
5. Confirm your site allows generated OOD session passwords to be displayed in `view.html.erb`.
6. Reload Open OnDemand app listings as required by your site deployment process.

### Open OnDemand security model in this wrapper

- Open WebUI listens on the OOD-assigned session port.
- Ollama listens on a randomized per-session backend port (`OLLAMA_BACKEND_PORT`) bound to localhost inside the job.
- Open WebUI connects to Ollama through that randomized backend URL only.
- A per-session Open WebUI admin password is generated by `template/before.sh.erb` and shown to the user in `view.html.erb`.
